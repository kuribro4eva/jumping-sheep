# Jumping Sheep GitHub Pages Publish Plan

Date: 2026-02-28
Scope: Migrate current single-file React artifact app to Vite React, publish to public GitHub repo `jumping-sheep`, deploy to GitHub Pages.

## Objectives

1. Keep current game behavior intact.
2. Replace Claude-specific storage API with browser `localStorage`.
3. Add fullscreen toggle for classroom projector usage.
4. Deploy at `https://<github-user>.github.io/jumping-sheep/`.

## Assumptions

1. GitHub CLI (`gh`) is authenticated for repo creation.
2. Node and npm are available locally.
3. Repo will be public and named exactly `jumping-sheep`.

## Work Plan

### Phase 1: Project Scaffolding

1. Initialize Vite React project files in repository root.
2. Place game component at `src/game.jsx`.
3. Update `src/App.jsx` to render `<JumpingSheep />`.
4. Remove unused starter assets/styles.

Deliverable:
- Vite app runs with `npm run dev` and builds with `npm run build`.

### Phase 2: Code Migration

1. Replace `window.storage.get/set` with `localStorage` load/save.
2. Add fullscreen state and event listener (`fullscreenchange`).
3. Add fullscreen toggle handler using `rootRef.current?.requestFullscreen?.()` and `document.exitFullscreen?.()`.
4. Add fullscreen button in header controls.
5. Add fullscreen CSS for `.game-root:fullscreen .game-frame`.

Deliverable:
- Gameplay parity maintained, storage persists locally, fullscreen toggles correctly.

### Phase 3: GitHub Pages Configuration

1. Set Vite base path to `/jumping-sheep/`.
2. Add GitHub Actions workflow for Pages build/deploy.
3. Ensure `dist` is generated by CI and published by Pages.

Deliverable:
- Automatic deployment on push to `main`.

### Phase 4: Repository Publish

1. Initialize git repository (if missing).
2. Commit project files.
3. Create and push public repo:
   - `gh repo create jumping-sheep --public --source=. --remote=origin --push`

Deliverable:
- Public GitHub repo exists and contains migrated project.

### Phase 5: Validation

1. Run `npm install`.
2. Run `npm run build`.
3. Open deployed URL and verify:
   - title/play flow
   - keyboard guessing
   - settings persistence
   - fullscreen toggle
   - win/loss overlays and navigation arrows

Deliverable:
- Deployment verified for classroom use.

## Risks and Mitigations

1. Risk: `gh` not authenticated.
   Mitigation: run `gh auth status`; if needed authenticate before create.
2. Risk: Base path mismatch breaks assets.
   Mitigation: enforce `base: '/jumping-sheep/'` and test deployed path.
3. Risk: Single-file migration introduces regressions.
   Mitigation: preserve existing `game.jsx` logic and only patch required sections.

## Success Criteria

1. Public repo `jumping-sheep` exists.
2. GitHub Pages site is live and reachable.
3. Game behavior matches handoff document.
